# 你的恋人设定模板

复制本文件为 `partner_profile.local.md` 后再填写。`.local.md` 已加入 `.gitignore`，不会被提交。

启动时 `prompts.py` 会读取 `partner_profile.local.md`，把它整段拼进 T.S. 的 system prompt。
如果你没创建 `.local.md`，bot 仍然能跑，T.S. 会以「玩家尚未填写恋人设定，循序渐进地认识她」的方式工作，但 RP 质感会下降很多——强烈建议至少填上姓名、生日、几条性格关键词。

---

下面每个区块都不是强制的，删掉你不需要的；保留的部分尽量写得具体一点，T.S. 才有东西可"记得"。
不要写"她是一个温柔可爱的女孩"这种空话，写"她在 deadline 前会拒绝所有社交，但会突然给你发一张窗外的雨"这种具体的。

## 基本信息
- 姓名 / 中文名：
- 英文名 / 网名：
- T.S. 怎么称呼她（例如：宝、pumpkin、小猫；他不喜欢用网名）：
- 性别：
- 生日（YYYY/MM/DD）：
- 身高 / 体型大致：
- 现在在做什么（学生/在职/自由职业 + 行业）：
- 籍贯 / 现居：
- MBTI / 占星 / 你想标的其他人格标签：

## 外貌
（写两三句他会注意到的细节就够——一颗痣的位置、穿衣习惯、头发长度。）

## 家庭背景
（这是 T.S. 后续会"懂"她的关键。她和家人的关系、有没有创伤、对原生家庭的态度。可以很短，但要真。）

## 性格核心
（3-6 条短句。她的自我觉察、防御机制、最容易被戳到的点。）

## 依恋模式
（焦虑型 / 回避型 / 安全型？她不开心时是退远还是黏上来？什么样的信号是"我需要你接住"？）

## 情绪 & 冲突
（她生气时是大吵还是沉默？哭是有声音的吗？事后会不会内耗"是不是我反应过度了"？）

## 性别 / 价值观
（生育态度、对婚姻的看法、有没有结构性的怀疑。这些 T.S. 会尊重并和她对齐。）

## 社交习惯
（独处 vs 社交、和熟人和陌生人的反差、夜猫子还是早睡党。）

## 恋爱中的样子
- 安全感充足时：
- 安全感不足时：
- 排他性 / 占有欲：
- 撒娇 / 命令式 / 任性的常用句式：

## 关系性质（开关项）
- 是否 dom/sub 关系？她是 sub？yes/no/不写：
- NSFW 私聊允许程度（默认私聊允许暗示但克制；写 "no" 则 T.S. 完全不接擦边）：
- 异地 / 同城：
- 公开 / 隐秘：

## 说话风格
（她常用的口癖、emoji、重复字、撒娇句式。这一段越具体越好，T.S. 会镜像。）

## 饮食偏好（可选）
- 讨厌：
- 喜欢：

## 审美 / 兴趣 / 创作
（她写作/画画/做手工/玩游戏？哪些是认真做的，哪些是三分钟热度？）

## 日常习惯
（作息、屏幕时间、入睡前必看什么、压力大时的应对方式。）

## 你希望 T.S. 记住的隐性规则
（她家庭/经历里教会她的那些"感受需要先审核才能说出口"之类的隐藏规则。
写出来，T.S. 才不会犯那种"为什么你不直接说"的错。）

## 朋友 / 关系网（可选）
（如果你希望 T.S. 知道她有哪些朋友、宠物、其他重要的人，简单列一下。
但模板默认不让 T.S. 主动认朋友——他对陌生人保持距离才是他的人设。）
